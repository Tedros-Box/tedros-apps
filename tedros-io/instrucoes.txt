## Prompt para o Antigravity

**Contexto e Objetivo**

Crie um website institucional e comercial para o projeto **Tedros**, hospedado no domínio **tedros.io**, com foco em **vendas B2B**, **apresentação executiva** e **demonstração técnica**.
O objetivo principal do site é convencer gestores de TI, gerentes de projeto e decisores técnicos (ex.: GlobalWeb, Detran e empresas similares) de que o **Tedros é uma solução estratégica baseada em IA para gestão de demandas, code review e automação inteligente de processos de desenvolvimento de software**.

O site deve comunicar claramente:

* Problemas reais do cenário atual
* A proposta de valor do Tedros
* Casos de uso práticos
* Diferenciais técnicos
* Arquitetura da solução
* Chamada clara para contato e demonstração

## Público-alvo

* Diretores e gerentes de TI
* Gerentes de projeto
* Arquitetos de software
* Líderes técnicos
* Empresas que utilizam Redmine, GitLab, JBoss, WSO2 e ecossistemas Java corporativos

## Contexto do Problema (a ser traduzido em narrativa de negócio)

O cliente (Detran) possui:

* Cerca de **200 sistemas Java** em produção
* Servidores de aplicação **JBoss EAP**
* Integração via **WSO2**
* Versionamento no **GitLab**
* Gestão de demandas no **Redmine**

Principais dores:

* Grande volume de demandas abertas no Redmine
* Processo rigoroso e demorado de aprovação de demandas
* Necessidade constante de **code review** nos merge requests
* Times distribuídos com múltiplos gerentes, desenvolvedores e analistas
* Dificuldade em **gerar e organizar evidências de trabalho**, exigidas para aprovação das demandas

---

## Proposta de Valor do Tedros

O **Tedros** é uma plataforma que:

* Utiliza **IA conversacional (Chatbot corporativo)** integrada aos sistemas do cliente
* Executa **tools/functions diretamente nos sistemas** (Redmine, GitLab, etc.)
* Permite:

  * Buscar, analisar e resumir issues do Redmine
  * Realizar **code review automatizado** em merge requests
  * Gerar análises sob demanda para gestores
  * Apoiar a tomada de decisão com base em dados reais
* Inclui um módulo para **gestão de evidências de trabalho**, permitindo:

  * Captura automática de screenshots durante o expediente
  * Seleção manual das evidências pelo colaborador
  * Associação das evidências às demandas do Redmine
  * Organização e rastreabilidade para auditoria e aprovação

1. **Home**

   * Chatbot com IA + tools executáveis

4. **Casos de Uso**

   * Análise de demandas no Redmine

5. **Arquitetura**

## Requisitos Técnicos Obrigatórios

* Java 17
* Projeto Maven do tipo **WAR**
* Arquitetura REST
* Deploy em **Apache TomEE com OpenEJB**
* Código organizado e extensível

* Layout moderno, corporativo e limpo
* Foco em conversão (vendas)
